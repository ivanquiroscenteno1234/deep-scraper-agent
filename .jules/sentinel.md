## 2024-05-24 - Path Traversal in Script Execution
**Vulnerability:** The `execute_script` API endpoint in `backend/main.py` allowed arbitrary python files to be executed by accepting an unvalidated `script_path` directly from the user request payload. An attacker could potentially pass relative paths (e.g., `../../sensitive_script.py`) to execute files outside the intended `output/generated_scripts` directory.
**Learning:** Even internal or admin-facing endpoints that execute files based on parameters need strict validation to prevent path traversal when passing inputs to `asyncio.create_subprocess_exec` or `subprocess.run`. Simple `os.path.exists()` checks do not prevent directory traversal.
**Prevention:** Always resolve the absolute path of the requested file using `os.path.abspath()` and verify it strictly starts with the authorized base directory's absolute path (including a trailing separator) before permitting execution or access.

## 2025-04-14 - Path Traversal via Stdout Injection
**Vulnerability:** The `/api/execute-script` endpoint extracted a CSV file path from the execution output (`stdout`) and blindly passed it to `open()` without resolving and validating that the file resided inside the intended `output/data/` directory. This could lead to Arbitrary File Read if a malicious script or an injected output stream contained `saved to ../../../../etc/passwd.csv`.
**Learning:** Data extracted from untrusted sources, including the `stdout` of dynamically generated scripts, must be treated as malicious user input. When using this data to construct file paths, strict directory confinement checks must be performed.
**Prevention:** Always use `os.path.abspath` to resolve any extracted paths and ensure they strictly start with the absolute path of the allowed base directory (using `startswith(os.path.join(allowed_dir, ""))`). As a safe fallback, extract the `basename` and construct a new safe path inside the allowed directory.

## 2025-04-21 - [Defense in Depth] FastAPI Input Validation
**Vulnerability:** FastAPI endpoints (`/api/run` and `/api/execute-script`) lacked strict input validation. While path traversal was prevented, the `search_query` had no length limits (DoS risk via large payload) and the dates lacked format validation. These raw inputs were eventually passed directly to subprocess executions.
**Learning:** Even when inputs are safely passed to `subprocess` execution (e.g. without `shell=True`), they should still be strictly validated as early as possible. This limits the application's attack surface against injection, DoS, and malformed data bugs.
**Prevention:** Always use Pydantic's `Field` with strict constraints (`max_length`, `pattern`) and strict types (`HttpUrl` instead of `str` for URLs) on all FastAPI route models to validate data at the application boundary.
